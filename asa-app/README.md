# SIA AI Literacy Survey App (ASA)

A web-based AI competency survey built for the **Singapore Institute of Architects (SIA)** to map AI adoption across Singapore's architectural practice.

**Live:** https://integrations-space.github.io/asa-app/

---

## What It Does

Participants scan a QR code, complete a 10-question survey (5–10 min), and receive a personalised AI competency report by email within 15 minutes.

The report includes:
- **Part 1 — Group Report:** cohort average, tier distribution, collective strengths and areas to improve
- **Part 2 — Individual Report:** personal score, question-by-question feedback, and tailored next steps

Reports are generated by an AI model (Groq / Gemini / Mistral / DeepSeek) and fall back to pre-written templates if no API key is configured.

---

## How It Works

```
Participant scans QR code
        ↓
Fills in registration + 10 questions
        ↓
React app scores answers & POSTs to Google Apps Script
        ↓
Apps Script writes row to Google Sheet (ASA_Responses)
        ↓
After 15 min → processReportQueue trigger fires
        ↓
AI generates Part 1 (group) + Part 2 (individual) report
        ↓
Gmail sends HTML email to participant
```

## Session Setup (for Facilitators)

1. Generate a unique Session ID: `ASA-{IDENTIFIER}-{DATE}` (e.g. `ASA-MORNING-01`)
2. Embed it in the QR code URL:
   ```
   https://integrations-space.github.io/asa-app/?sessionId=ASA-MORNING-01&groupName=SIA%20Session%201
   ```
3. All responses from that cohort are tagged with the Session ID in the Google Sheet

## Configuration

All survey content (questions, registration fields, branding, tiers) is in one file:
`src/config/survey.config.js`

The backend URL is stored as a GitHub Actions secret (`VITE_BACKEND_URL`) and injected at build time — never committed to the repo.

---

## Local Development

```bash
npm install
npm run dev
```

To test with the live backend, create `.env` in the project root:
```
VITE_BACKEND_URL=https://script.google.com/macros/s/.../exec
```
